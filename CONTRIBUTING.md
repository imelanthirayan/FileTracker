# Contributing to File Tracker

Thank you for your interest in contributing!

## How to Contribute

- Fork the repository and create a new branch for your feature or bugfix.
- Write clear, concise commit messages.
- Add or update tests for your changes.
- Ensure all code passes linting and tests before submitting a pull request.
- Describe your changes and reference any related issues in your PR.

## Code Style
- Use type hints where possible.
- Follow PEP8 for Python code.
- Keep CLI help messages clear and descriptive.

## Reporting Issues
- Please include steps to reproduce, expected behavior, and actual behavior.
- Attach relevant logs or screenshots if possible.

## Questions?
Open an issue or discussion in the repository.
